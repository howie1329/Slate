# Slate agent guide

Slate is a local-first macOS task planner built around a daily commitment budget. The current app provides the compact desktop shell and the `/today` and `/inbox` routes; task-domain logic, persistence, menu-bar behavior, and AI are intentionally deferred.

## Project map

- `src/routes/`: file-based TanStack Router routes and page UI.
- `src/routes/__root.tsx`: shared Slate shell and navigation.
- `src/router.tsx`: router setup; `src/routeTree.gen.ts` is generated by the router plugin.
- `src/styles.css`: Tailwind entry point and global font tokens.
- `src-tauri/`: Rust host, Tauri configuration, capabilities, and native bundle assets.
- `docs/`: project documentation; add implementation plans under `docs/plans/`.

## Commands

Use npm with the committed `package-lock.json`.

| Purpose | Command |
| --- | --- |
| Run the web dev server | `npm run dev` |
| Build the web application and type-check | `npm run build` |
| Preview the web build | `npm run preview` |
| Run a Tauri subcommand | `npm run tauri -- <subcommand>` |

For local desktop development, use `npm run tauri -- dev`. For a native release bundle, use `npm run tauri -- build`.

## Working rules

- Keep the task workflow deterministic and local-first unless the work explicitly expands that scope.
- Add routes as files in `src/routes/`; do not hand-edit `src/routeTree.gen.ts`.
- Keep native behavior and permissions scoped to `src-tauri/`; review Tauri capabilities and configuration when exposing host functionality.
- Preserve the compact-window experience defined in `src-tauri/tauri.conf.json` when changing layout.
- Follow `DESIGN.md` for UI work, `CODE-QUALITY.md` for maintainability, and `CHANGE-EXPLANATION.md` when handing work off.
- Do not introduce a testing command into documentation until one is configured in `package.json`.

## Before finishing

1. Run the relevant real command from the table above when the change warrants validation.
2. Confirm route changes did not require edits to generated router output.
3. Check that desktop-specific changes remain usable within the configured minimum window size.
4. Summarize user-visible behavior, changed files, and validation performed.
